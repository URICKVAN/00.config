# =========================================================
# CONVENTIONAL COMMITS QUICK GUIDE
# =========================================================
#
# Format:
# <type>(<optional-scope>): <short summary>
#
# Summary rules:
# - Use imperative mood ("add", not "added")
# - Max 50 characters
# - Do not end with a period
#
# ---------------------------------------------------------
# TYPE REFERENCE
#
# feat     → new feature
# fix      → bug fix
# docs     → documentation changes
# style    → formatting only (no logic change)
# refactor → internal restructuring (no behavior change)
# perf     → performance improvements
# test     → adding or modifying tests
# chore    → maintenance tasks (no production impact)
# build    → build system or dependencies
# ci       → CI/CD related changes
#
# ---------------------------------------------------------
# BODY (optional)
#
# Explain:
# - What changed
# - Why it changed
# - Context if necessary
#
# Wrap lines at 72 characters.
#
# ---------------------------------------------------------
# BREAKING CHANGES (if applicable)
#
# Use:
# BREAKING CHANGE: describe the impact
#
# Example:
# feat(auth): add JWT authentication support
#
# Adds token-based authentication using PyJWT.
# Improves API security for external integrations.
#
# BREAKING CHANGE: login endpoint response format updated
#
# ---------------------------------------------------------
# REFERENCES (optional)
#
# Closes #123
# Related to #456
#
# =========================================================
